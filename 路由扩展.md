# 路由器 (Router)

OTD Core 的路由器模块提供了增强的 Vue Router 功能，核心特性是基于文件系统的虚拟路由插件。

## 虚拟路由插件

基于文件系统自动生成路由配置的 Vite 插件，支持约定式路由。

### 核心特性

- **文件系统路由**: 基于 `src/pages` 目录结构自动生成路由
- **动态路由支持**: 支持 `[id].vue` 格式的动态参数路由
- **嵌套路由**: 支持目录嵌套的子路由结构
- **热更新**: 开发时文件变化自动更新路由

### 插件配置

```javascript
// vite.config.js
import { createVirtualRoutePlugin } from '@x-ui/otd-core';

export default {
  plugins: [
    createVirtualRoutePlugin({
      pagesDir: 'src/pages', // 页面目录，默认 'src/pages'
      enableCache: true // 是否启用缓存，默认 true
    })
  ]
};
```

### 路由生成规则

虚拟路由插件会根据文件系统结构自动生成对应的路由配置：

#### 基础路由

```
src/pages/
├── index.vue          → /                (根路由)
├── about.vue          → /about
├── contact.vue        → /contact
└── help.vue           → /help
```

#### 嵌套路由

```
src/pages/
├── index.vue          → /
├── user/
│   ├── index.vue      → /user           (父路由)
│   ├── profile.vue    → /user/profile   (子路由)
│   └── settings.vue   → /user/settings  (子路由)
└── admin/
    ├── index.vue      → /admin          (父路由)
    ├── dashboard.vue  → /admin/dashboard
    └── users.vue      → /admin/users
```

#### 动态路由

```
src/pages/
├── user/
│   ├── index.vue      → /user
│   ├── [id].vue       → /user/:id       (动态参数)
│   └── [id]/
│       ├── edit.vue   → /user/:id/edit
│       └── posts/
│           └── [postId].vue → /user/:id/posts/:postId
└── blog/
    ├── index.vue      → /blog
    └── [slug].vue     → /blog/:slug
```

#### 复杂路由结构示例

```
src/pages/
├── index.vue                    → /
├── about.vue                    → /about
├── user/
│   ├── index.vue                → /user
│   ├── profile.vue              → /user/profile
│   ├── [id].vue                 → /user/:id
│   └── [id]/
│       ├── edit.vue             → /user/:id/edit
│       ├── posts/
│       │   ├── index.vue        → /user/:id/posts
│       │   └── [postId].vue     → /user/:id/posts/:postId
│       └── settings/
│           ├── index.vue        → /user/:id/settings
│           ├── profile.vue      → /user/:id/settings/profile
│           └── security.vue     → /user/:id/settings/security
├── admin/
│   ├── index.vue                → /admin
│   ├── dashboard.vue            → /admin/dashboard
│   ├── users/
│   │   ├── index.vue            → /admin/users
│   │   ├── create.vue           → /admin/users/create
│   │   └── [userId]/
│   │       ├── index.vue        → /admin/users/:userId
│   │       └── edit.vue         → /admin/users/:userId/edit
│   └── settings/
│       ├── index.vue            → /admin/settings
│       ├── general.vue          → /admin/settings/general
│       └── security.vue         → /admin/settings/security
└── blog/
    ├── index.vue                → /blog
    ├── categories/
    │   ├── index.vue            → /blog/categories
    │   └── [category].vue       → /blog/categories/:category
    └── [slug]/
        ├── index.vue            → /blog/:slug
        └── comments.vue         → /blog/:slug/comments
```

### 路由器集成

```javascript
// router/index.js
import routes from 'virtual:routes';
import { createAppRouter, createWebHistory } from '@x-ui/otd-core';

const router = createAppRouter({
  history: createWebHistory(),
  routes,
  enableAuthGuard: true // 可选：启用认证守卫
});

export default router;
```

### 自动组件命名

虚拟路由插件会自动为组件生成名称：

```javascript
// 文件: src/pages/user/profile.vue
// 生成组件名: UserProfile
// 生成路由名: userProfile

// 文件: src/pages/admin/[id].vue
// 生成组件名: AdminId
// 生成路由名: adminId

// 文件: src/pages/blog/[slug]/comments.vue
// 生成组件名: BlogSlugComments
// 生成路由名: blogSlugComments
```

### 空文件处理

插件会自动为空的 `.vue` 文件添加基础模板：

```vue
<template>
  <div></div>
</template>
```

### 特殊目录

- `components` 目录会被自动忽略，不会生成路由
- 只有 `.vue` 文件会被处理为路由组件

## 认证守卫

OTD Core 提供了完整的认证守卫系统，支持 SSO 单点登录和权限控制。

### 功能特性

- **SSO 集成**: 支持基于 ticket 的 SSO 自动登录
- **权限检查**: 检查用户应用访问权限
- **资源加载**: 自动加载应用所需资源
- **URL 清理**: 自动清理认证参数，保持 URL 整洁

### 认证流程

认证守卫按以下流程执行检查：

```
                        ┌──────────────────┐
                        │ Route Navigation │
                        └─────────┬────────┘
                                  │
                                  ▼
                            ┌──────────┐    Yes     ┌──────────────┐
                            │ Public?  ├───────────►│ Allow Access │
                            └─────┬────┘            └──────┬───────┘
                                  │No                      │
                                  ▼                        │
                            ┌──────────────┐    Yes        │
                            │Authenticated?├─────────┐     │
                            └─────┬────────┘         │     │
                                  │No                │     │
                                  ▼                  │     │
                            ┌──────────┐             │     │
                            │HasTicket?│             ▼     ▼
                            └─────┬────┘     ┌─────────────┬─────────────┐
                                  │          │Check Access │   Render    │
                                  │          │ Permission  │    Page     │
                         ┌────────┴────────┐ └─────────┬───┘─────────────┘
                    No   │                 │Yes        │
                         ▼                 ▼           ▼
                ┌─────────┐        ┌──────────┐ ┌──────────┐
                │Redirect │        │Try SSO   │ │Load App  │
                │ Login   │        │ Login    │ │Resources │
                └────┬────┘        └─────┬────┘ └─────┬────┘
                     │                   │Fail        │Success
                     │                   ▼            ▼
                     │             ┌──────────┐ ┌──────────┐
                     └─────────────┤SSO Failed│ │Clean URL │
                                   └──────────┘ └─────┬────┘
                                                      │
                                                      ▼
                                                ┌──────────┐
                                                │Allow Page│
                                                └──────────┘
```

详细步骤说明：

1. **路径检查**: 检查目标路径是否为公开路径
2. **认证状态**: 检查用户是否已认证
3. **SSO 登录**: 如果 URL 中包含 ticket 参数，尝试 SSO 自动登录
4. **权限验证**: 检查用户是否有应用访问权限
5. **资源加载**: 加载应用所需的资源（如用户信息、权限等）
6. **路由跳转**: 根据检查结果允许访问或重定向到登录页

### SSO 单点登录

当用户通过 SSO 系统访问应用时，URL 中会包含 ticket 参数：

```
https://your-app.com/dashboard?ticket=ST-1234567890
```

认证守卫会自动：

1. 提取 ticket 参数
2. 调用 SSO 登录接口验证票据
3. 登录成功后清理 URL 中的 ticket 参数
4. 继续正常的路由导航

### 自动添加的公共路由

启用认证守卫的路由器会自动添加以下公共路由：

| 路径               | 名称      | 组件      | 说明         |
| ------------------ | --------- | --------- | ------------ |
| `/login`           | Login     | Login.vue | 登录页面     |
| `/403`             | Forbidden | 403.vue   | 权限不足页面 |
| `/:pathMatch(.*)*` | NotFound  | 404.vue   | 404 页面     |

## API 参考

### 插件选项

```typescript
interface VirtualRoutePluginOptions {
  pagesDir?: string; // 页面目录，默认 'src/pages'
  enableCache?: boolean; // 是否启用缓存，默认 true
}
```

## 注意事项

1. **目录结构**: 确保 `src/pages` 目录存在并正确组织
2. **文件命名**: 使用 `.vue` 扩展名，支持动态路由 `[param].vue` 格式
3. **组件目录**: `components` 目录会被自动忽略，不会生成路由
4. **index.vue**: 作为目录的默认路由，支持嵌套路由结构
