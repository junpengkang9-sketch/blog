# 代码规范

## 概述

本项目采用 ESLint + Prettier 的代码规范体系，配合 VSCode 编辑器设置，确保代码风格一致性和质量。

## ESLint 配置

### 基础规则

项目使用 ESLint 9.x 的 Flat Config 格式，配置文件为 `eslint.config.mjs`。

#### 忽略文件

- `node_modules/**` - 依赖目录
- `dist/**` - 构建产物

#### 环境配置

- ECMAScript 版本：latest
- 模块类型：ES6 modules
- 全局变量：browser + node + es2020

### 导入规则

#### 导入排序

```javascript
// 导入顺序：builtin → external → internal → parent → sibling → index
import fs from 'fs'; // builtin
import Vue from 'vue'; // external
import { utils } from '@/utils'; // internal
import config from '../config'; // parent
import './style.css'; // sibling
import component from './index.vue'; // index
```

#### 导入规范

**导入后必须换行**

```javascript
// ✅ 正例
import Vue from 'vue';

const app = createApp();

// ❌ 反例
import Vue from 'vue';
const app = createApp();
```

**禁止重复导入同一模块**

```javascript
// ✅ 正例
import { ref, reactive, computed } from 'vue';

// ❌ 反例
import { ref } from 'vue';
import { reactive } from 'vue';
import { computed } from 'vue';
```

**组间必须空行，按字母顺序排序**

```javascript
// ✅ 正例
import fs from 'fs';
import path from 'path';

import axios from 'axios';
import Vue from 'vue';

import { utils } from '@/utils';
import config from '@/config';

import helper from '../helper';
import validator from '../validator';

// ❌ 反例
import Vue from 'vue';
import fs from 'fs';
import { utils } from '@/utils';
import axios from 'axios';
```

**import 内部成员按字母顺序排序**

```javascript
// ✅ 正例
import { computed, onMounted, reactive, ref } from 'vue';

// ❌ 反例
import { ref, onMounted, computed, reactive } from 'vue';
```

### 代码质量规则

#### 现代 JavaScript

**`no-var`: 禁用 var，使用 let/const**

```javascript
// ✅ 正例
const name = 'John';
let age = 25;

// ❌ 反例
var name = 'John';
var age = 25;
```

**`prefer-const`: 优先使用 const**

```javascript
// ✅ 正例
const users = ['Alice', 'Bob'];
const config = { api: '/api/v1' };

// ❌ 反例
let users = ['Alice', 'Bob']; // 值不会改变
let config = { api: '/api/v1' }; // 值不会改变
```

**`prefer-template`: 优先使用模板字符串**

```javascript
// ✅ 正例
const message = `Hello, ${name}!`;
const url = `${baseUrl}/users/${userId}`;

// ❌ 反例
const message = 'Hello, ' + name + '!';
const url = baseUrl + '/users/' + userId;
```

**`object-shorthand`: 使用对象属性简写**

```javascript
// ✅ 正例
const obj = {
  name,
  getValue() {
    return this.value;
  },
  [key]: value
};

// ❌ 反例
const obj = {
  name: name,
  getValue: function () {
    return this.value;
  }
};
```

#### 安全性

**`no-eval`: 禁用 eval()**

```javascript
// ✅ 正例
const result = JSON.parse(jsonString);
const fn = new Function('return 42');

// ❌ 反例
const result = eval(code);
const value = eval('1 + 1');
```

**`no-implied-eval`: 禁用隐式 eval()**

```javascript
// ✅ 正例
setTimeout(() => {
  console.log('Hello');
}, 1000);
setInterval(() => {
  doSomething();
}, 1000);

// ❌ 反例
setTimeout('console.log("Hello")', 1000);
setInterval('doSomething()', 1000);
```

**`no-new-func`: 禁用 Function 构造函数**

```javascript
// ✅ 正例
const add = (a, b) => a + b;
function multiply(a, b) {
  return a * b;
}

// ❌ 反例
const add = new Function('a', 'b', 'return a + b');
```

**`no-script-url`: 禁用 javascript: URLs**

```javascript
// ✅ 正例
<a href="#" onClick={handleClick}>Click me</a>
<button onClick={handleSubmit}>Submit</button>

// ❌ 反例
<a href="javascript:void(0)">Click me</a>
<a href="javascript:alert('Hello')">Alert</a>
```

#### 错误处理

**`default-case`: switch 语句需要 default 分支**

```javascript
// ✅ 正例
switch (type) {
  case 'success':
    return 'green';
  case 'error':
    return 'red';
  default:
    return 'gray';
}

// ❌ 反例
switch (type) {
  case 'success':
    return 'green';
  case 'error':
    return 'red';
  // 缺少 default 分支
}
```

**`no-empty`: 禁用空代码块**

```javascript
// ✅ 正例
if (condition) {
  doSomething();
}

try {
  riskyOperation();
} catch (error) {
  console.error('Error occurred:', error);
}

// ❌ 反例
if (condition) {
  // 空代码块
}

try {
  riskyOperation();
} catch (error) {
  // 空的 catch 块
}
```

**`no-fallthrough`: 禁用 switch 穿透**

```javascript
// ✅ 正例
switch (value) {
  case 1:
    doFirst();
    break;
  case 2:
    doSecond();
    break;
  case 3:
  // 明确标注的穿透
  // eslint-disable-next-line no-fallthrough
  case 4:
    doThirdOrFourth();
    break;
}

// ❌ 反例
switch (value) {
  case 1:
    doFirst();
  // 意外穿透到下一个 case
  case 2:
    doSecond();
    break;
}
```

**`no-unreachable`: 禁用不可达代码**

```javascript
// ✅ 正例
function process(data) {
  if (!data) {
    return null;
  }
  return processData(data);
}

// ❌ 反例
function process(data) {
  return data;
  console.log('This will never execute'); // 不可达代码
}
```

#### 命名约定

**`camelcase`: 使用驼峰命名（属性除外）**

```javascript
// ✅ 正例
const userName = 'john';
const isLoggedIn = true;
const { user_id } = response; // 解构中允许

// ❌ 反例
const user_name = 'john';
const is_logged_in = true;
```

**`new-cap`: 构造函数首字母大写**

```javascript
// ✅ 正例
class UserManager {}
const user = new UserManager();
const date = new Date();

// ❌ 反例
class userManager {}
const user = new userManager();
function userManager() {}
const user = new userManager();
```

**允许特定下划线变量**

```javascript
// ✅ 正例 - 特殊变量允许下划线
const _id = user._id;
const currentDir = __dirname;
const filename = __filename;

// ❌ 反例 - 普通变量不允许下划线
const _privateVar = 'secret';
const user_data = {};
```

#### 性能优化

**`no-array-constructor`: 禁用 Array 构造函数**

```javascript
// ✅ 正例
const items = [];
const numbers = [1, 2, 3, 4, 5];
const empty = new Array(length).fill(0); // 明确填充值时允许

// ❌ 反例
const items = new Array();
const numbers = new Array(1, 2, 3, 4, 5);
```

**`no-loop-func`: 禁用循环中的函数**

```javascript
// ✅ 正例
const handlers = [];
for (let i = 0; i < 10; i++) {
  handlers.push(createHandler(i));
}

function createHandler(index) {
  return () => console.log(index);
}

// ❌ 反例
const handlers = [];
for (let i = 0; i < 10; i++) {
  handlers.push(() => console.log(i)); // 闭包陷阱
}
```

**`no-new-object`: 禁用 Object 构造函数**

```javascript
// ✅ 正例
const obj = {};
const user = { name: 'John', age: 30 };
const config = Object.create(null);

// ❌ 反例
const obj = new Object();
const user = new Object({ name: 'John', age: 30 });
```

#### 代码维护性

**`complexity`: 圈复杂度不超过 10**

```javascript
// ✅ 正例 - 圈复杂度低
function validateUser(user) {
  if (!user) return false;
  if (!user.name) return false;
  if (!user.email) return false;
  return true;
}

// ❌ 反例 - 圈复杂度过高
function processData(data, type, options, mode) {
  if (type === 'user') {
    if (mode === 'create') {
      if (options.validate) {
        if (data.name && data.email) {
          // 多层嵌套增加复杂度
        }
      }
    } else if (mode === 'update') {
      // 更多分支...
    }
  } else if (type === 'order') {
    // 更多逻辑...
  }
}
```

**`max-depth`: 最大嵌套深度 4 层**

```javascript
// ✅ 正例 - 提取函数减少嵌套
function processUser(user) {
  if (!user) return;

  return validateAndProcess(user);
}

function validateAndProcess(user) {
  if (!user.name) return false;
  if (!user.email) return false;
  return processUserData(user);
}

// ❌ 反例 - 嵌套过深
function processUser(user) {
  if (user) {
    if (user.name) {
      if (user.email) {
        if (user.age > 18) {
          if (user.verified) {
            // 嵌套太深，难以维护
          }
        }
      }
    }
  }
}
```

**`max-lines-per-function`: 单函数最大 100 行**

```javascript
// ✅ 正例 - 拆分大函数
function createUser(userData) {
  const validatedData = validateUserData(userData);
  const formattedData = formatUserData(validatedData);
  return saveUser(formattedData);
}

function validateUserData(userData) {
  // 验证逻辑...
}

function formatUserData(userData) {
  // 格式化逻辑...
}

// ❌ 反例 - 单函数过长
function createUser(userData) {
  // 100+ 行代码在一个函数中
  // 验证、格式化、保存等逻辑混在一起
}
```

**`max-params`: 函数参数不超过 4 个**

```javascript
// ✅ 正例 - 使用对象参数
function createUser(options) {
  const { name, email, age, department } = options;
  // 实现逻辑...
}

createUser({
  name: 'John',
  email: 'john@example.com',
  age: 30,
  department: 'IT'
});

// ❌ 反例 - 参数过多
function createUser(name, email, age, department, role, status, permissions) {
  // 参数太多，难以维护
}
```

### Vue 专用规则

#### 组件命名

**`vue/component-name-in-template-casing`: 模板中使用 PascalCase**

```vue
<!-- ✅ 正例 -->
<template>
  <UserCard :user-data="user" />
  <ProductList :items="products" />
  <MyCustomButton @click="handleClick" />
</template>

<!-- ❌ 反例 -->
<template>
  <user-card :user-data="user" />
  <product-list :items="products" />
  <my-custom-button @click="handleClick" />
</template>
```

**`vue/attribute-hyphenation`: 属性使用 kebab-case**

```vue
<!-- ✅ 正例 -->
<template>
  <UserCard :user-data="user" :is-loading="loading" :max-items="100" />
</template>

<!-- ❌ 反例 -->
<template>
  <UserCard :userData="user" :isLoading="loading" :maxItems="100" />
</template>
```

**`vue/prop-name-casing`: props 使用 camelCase**

```vue
<!-- ✅ 正例 -->
<script setup>
const props = defineProps({
  userData: Object,
  isLoading: Boolean,
  maxItems: Number
});
</script>

<!-- ❌ 反例 -->
<script setup>
const props = defineProps({
  'user-data': Object,
  'is-loading': Boolean,
  'max-items': Number
});
</script>
```

**`vue/custom-event-name-casing`: 事件使用 kebab-case**

```vue
<!-- ✅ 正例 -->
<script setup>
const emit = defineEmits(['user-click', 'item-selected', 'data-updated']);

function handleUserClick() {
  emit('user-click', user);
}
</script>

<!-- ❌ 反例 -->
<script setup>
const emit = defineEmits(['userClick', 'itemSelected', 'dataUpdated']);

function handleUserClick() {
  emit('userClick', user);
}
</script>
```

#### Composition API

**`vue/define-emits-declaration`: 使用类型字面量声明 emits**

```vue
<!-- ✅ 正例 -->
<script setup lang="ts">
const emit = defineEmits<{
  userClick: [user: User];
  itemSelected: [id: string, item: Item];
  close: [];
}>();
</script>

<!-- ❌ 反例 -->
<script setup>
const emit = defineEmits({
  userClick: null,
  itemSelected: null,
  close: null
});
</script>
```

**`vue/define-props-declaration`: 使用基于类型的 props 声明**

```vue
<!-- ✅ 正例 -->
<script setup lang="ts">
interface Props {
  user: User;
  isLoading?: boolean;
  items: Item[];
}

const props = defineProps<Props>();
// 或者
const props = defineProps<{
  user: User;
  isLoading?: boolean;
  items: Item[];
}>();
</script>

<!-- ❌ 反例 -->
<script setup>
const props = defineProps({
  user: Object,
  isLoading: Boolean,
  items: Array
});
</script>
```

**`vue/no-boolean-default`: 布尔 props 默认值为 false**

```vue
<!-- ✅ 正例 -->
<script setup lang="ts">
interface Props {
  isVisible?: boolean; // 默认 false
  isLoading?: boolean; // 默认 false
}

const props = withDefaults(defineProps<Props>(), {
  isVisible: false,
  isLoading: false
});
</script>

<!-- ❌ 反例 -->
<script setup lang="ts">
const props = withDefaults(
  defineProps<{
    isVisible?: boolean;
  }>(),
  {
    isVisible: true // 不应该默认为 true
  }
);
</script>
```

**`vue/prefer-define-options`: 优先使用 defineOptions**

```vue
<!-- ✅ 正例 -->
<script setup>
defineOptions({
  name: 'UserCard',
  inheritAttrs: false
});

const props = defineProps(['user']);
</script>

<!-- ❌ 反例 -->
<script>
export default {
  name: 'UserCard',
  inheritAttrs: false
};
</script>

<script setup>
const props = defineProps(['user']);
</script>
```

```vue
<!-- ✅ 正例 -->
<template>
```

**`vue/no-static-inline-styles`: 避免静态内联样式**

```vue
<template>
```

**`vue/no-useless-mustaches`: 避免无用的 mustache 语法**

```vue
  </div>

```

**`vue/prefer-separate-static-class`: 静态 class 单独声明**

```vue
<!-- ✅ 正例 -->
<template>

</template>

<!-- ❌ 反例 -->
<template>
  <div :class="['card', 'base-style', { active: isActive }]">Content</div>
</template>
```

## Prettier 配置

### 基础格式化

```javascript
// .prettierrc.mjs
export default {
  printWidth: 100, // 行宽 100 字符
  tabWidth: 2, // 缩进 2 空格
  useTabs: false, // 使用空格缩进
  semi: true, // 语句末尾加分号

  quoteProps: 'as-needed', // 对象属性按需加引号
```

### Vue 特定配置

- `vueIndentScriptAndStyle`: false - script/style 不额外缩进
- `singleAttributePerLine`: false - HTML 属性不强制每行一个
- `htmlWhitespaceSensitivity`: 'css' - 遵循 CSS 空格规则

## StyleLint 配置

### 基础配置

项目使用 StyleLint 进行样式代码规范检查，配置文件为 `stylelint.config.mjs`。

#### 扩展配置

- `stylelint-config-standard` - StyleLint 官方标准配置
- `stylelint-config-standard-scss` - SCSS 标准配置
- `stylelint-config-recess-order` - CSS 属性自动排序

#### 插件

- `@stylistic/stylelint-plugin` - 样式格式化插件
- `stylelint-selector-bem-pattern` - BEM 命名检查插件

### 命名规范

#### BEM 命名规范

**`selector-class-pattern`: 类名支持 kebab-case 和 BEM 格式**

```scss
/* ✅ 正例 - kebab-case */
.page-wrapper {
  display: flex;
}

.user-card {
  padding: 1rem;
}

/* ✅ 正例 - BEM 格式 */
.block {
  background: #fff;
}

.block__element {
  color: #333;
}

.block--modifier {
  font-weight: bold;
}

.block__element--modifier {
  text-decoration: underline;
}

/* ✅ 正例 - kebab-case + BEM 混合 */
.user-card__header--active {
  background: #007bff;
}

/* ❌ 反例 - 不符合规范 */
.UserCard {  /* 不允许大驼峰 */
  padding: 1rem;
}

.user_card {  /* 不允许下划线（BEM 元素除外）*/
  padding: 1rem;
}

.page——wrapper {  /* 使用了中文破折号 */
  display: flex;
}
```

#### 禁止 ID 选择器

**`selector-max-id`: 不允许使用 ID 选择器**

```scss
/* ✅ 正例 */
.header {
  background: #333;
}

.nav-menu {
  display: flex;
}

/* ❌ 反例 */
#header {
  background: #333;
}

#navMenu {
  display: flex;
}
```

#### 禁止重复属性

```scss
/* ✅ 正例 */
```

### 样式格式化规则

#### 缩进规则

**`@stylistic/indentation`: 使用 2 个空格缩进**

```scss
/* ✅ 正例 */
.card {
  padding: 1rem;

  .header {
    font-size: 1.2rem;
  }
}

/* ❌ 反例 */
.card {
    padding: 1rem;  /* 4 空格缩进 */

    .header {
        font-size: 1.2rem;
    }
}
```

#### 引号规则

**`@stylistic/string-quotes`: 字符串必须使用单引号**

```scss
/* ✅ 正例 */
.card {
  content: 'Hello';
  font-family: 'Arial', sans-serif;
  background-image: url('/images/bg.jpg');
}

/* ❌ 反例 */
.card {
  content: "Hello";  /* 双引号 */
  font-family: "Arial", sans-serif;
  background-image: url("/images/bg.jpg");
}
```

#### 伪元素规则

**`selector-pseudo-element-colon-notation`: 伪元素必须使用双冒号**

```scss
/* ✅ 正例 */
.card::before {
  content: '';
}

.card::after {
  content: '';
}

/* ❌ 反例 */
.card:before {  /* 单冒号 */
  content: '';
}

.card:after {
  content: '';
}
```

#### 零值单位

**`length-zero-no-unit`: 数值为 0 时不需要单位**

```scss
/* ✅ 正例 */
.card {
  margin: 0;
  padding: 0;
  border: 0;
}

/* ❌ 反例 */
.card {
  margin: 0px;  /* 0 不需要单位 */
  padding: 0rem;
  border: 0em;
}
```

### CSS 属性排序

**`stylelint-config-recess-order`: 自动按照特定顺序排列 CSS 属性**

属性排序规则：定位 → 盒模型 → 文本 → 视觉 → 其他

```scss
/* ✅ 正例 - 属性按顺序排列 */
.card {
  /* 定位 */
  position: relative;
  top: 0;
  left: 0;
  z-index: 10;

  /* 盒模型 */
  display: flex;
  width: 100%;
  height: 200px;
  padding: 1rem;
  margin: 0 auto;

  /* 文本 */
  font-size: 1rem;
  line-height: 1.5;
  color: #333;

  /* 视觉 */
  background: #fff;
  border: 1px solid #ddd;
  border-radius: 4px;
}

/* ❌ 反例 - 属性顺序混乱 */
.card {
  color: #333;
  position: relative;
  background: #fff;
  display: flex;
  font-size: 1rem;
  width: 100%;
}
```

### 文件类型配置

#### Vue 文件

使用 `postcss-html` 解析器处理 `.vue` 文件中的样式

```vue
<template>
  <div class="user-card">
    {{ user.name }}
  </div>
</template>

<style scoped>
.user-card {
  padding: 1rem;
  background: #fff;
}
</style>
```

#### SCSS 文件

使用 `postcss-scss` 解析器处理 `.scss` 文件

```scss
// common.scss
.page-wrapper {
  display: flex;
  flex-direction: column;
  height: 100%;
}
```

## VSCode 配置

### 编辑器设置

#### 格式化配置

```json
{
  "editor.formatOnSave": true, // 保存时自动格式化
  "editor.defaultFormatter": "esbenp.prettier-vscode", // 默认使用 Prettier
  "prettier.requireConfig": true, // 要求 Prettier 配置文件
  "prettier.useEditorConfig": false // 不使用 EditorConfig
}
```

#### ESLint 集成

```json
{
  "eslint.enable": true, // 启用 ESLint
  "eslint.useFlatConfig": true, // 使用 Flat Config
  "eslint.validate": [
    // 验证文件类型
    "javascript",
    "javascriptreact",
    "typescript",
    "typescriptreact",
    "vue",
    "html",
    "markdown"
  ]
}
```

#### 保存时操作

```json
{
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit", // 自动修复 ESLint 错误
    "source.fixAll.stylelint": "explicit" // 自动修复 StyleLint 错误
  },
  "files.trimTrailingWhitespace": true, // 清理行尾空格
  "files.trimFinalNewlines": true, // 清理文件末尾空行
  "files.insertFinalNewline": true // 文件末尾插入换行
}
```
