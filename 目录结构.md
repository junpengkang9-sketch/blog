# 目录结构

本文档介绍基于 Vue 3 + Vite 的前端项目标准目录结构和文件组织规范。

## 项目根目录结构

```
npm-template-ui/
├── .vscode/                 # VS Code 工作区配置
├── public/                  # 静态资源目录
├── src/                     # 源代码目录
├── .gitattributes          # Git 文件属性配置(统一换行符为 LF)
├── .gitignore              # Git 忽略文件配置
├── .npmrc                  # npm 配置文件
├── .prettierrc.mjs         # Prettier 代码格式化配置
├── eslint.config.mjs       # ESLint 代码规范配置
├── index.html              # 入口 HTML 文件
├── package.json            # 项目依赖和脚本配置
├── package-lock.json       # 依赖版本锁定文件
├── README.md               # 项目说明文档
├── vite.config.js          # Vite 构建工具配置
└── wormhole.js             # Wormhole 配置文件(Swagger API 自动生成)
```

## 目录命名规范

### 文件夹命名

- 使用 **kebab-case** (短横线命名法)
- 英文小写，单词间用 `-` 连接
- 语义化命名，体现功能用途

```bash
# ✅ 正确示例
components/
user-profile/
api-services/

# ❌ 错误示例
Components/
userProfile/
APIServices/
```

### 文件命名

- **Vue 组件**: 使用 **PascalCase** 或 **kebab-case**
- **JavaScript 文件**: 使用 **camelCase** 或 **kebab-case**
- **样式文件**: 使用 **kebab-case**

```bash
# ✅ Vue 组件命名
UserProfile.vue
user-profile.vue

# ✅ JavaScript 文件命名
userService.js
user-service.js

# ✅ 样式文件命名
main-styles.scss
user-profile.scss
```

## 文件组织原则

### 1. 按功能模块组织

相关文件放在同一目录下，便于维护和查找。页面模块主要包含页面组件和专用组件，而 services 和 store 在 src/ 根目录下按模块统一维护。

```
pages/user/
├── index.vue              # 用户列表页
├── profile.vue            # 用户详情页
└── components/            # 用户模块专用组件
    ├── UserCard.vue
    └── UserForm.vue

services/
├── index.js               # 服务导出
└── modules/
    ├── userService.js     # 用户相关 API
    └── authService.js     # 认证相关 API

store/
├── index.js               # Store 配置入口
└── modules/
    ├── userStore.js       # 用户状态模块
    └── authStore.js       # 认证状态模块
```

### 2. 分层架构

按照功能层次组织代码，保持清晰的依赖关系。

```
数据层 (services) → 状态层 (store) → 组件层 (components) → 页面层 (pages)
```

### 3. 索引文件

每个目录使用 `index.js` 作为导出入口。

```javascript
// components/index.js
export { default as UserCard } from './UserCard.vue';
export { default as UserForm } from './UserForm.vue';

// 使用时
import { UserCard, UserForm } from '@/components';
```

## 特殊目录说明

### assets vs public

- **assets/**: 需要被构建工具处理的资源 (图片压缩、路径处理等)
- **public/**: 直接复制到输出目录的静态资源

### components 分类

- **common/**: 纯 UI 组件，无业务逻辑
- **business/**: 包含业务逻辑的组件
- **layout/**: 页面布局相关组件
- **helper-select/**: 注册公共的辅助选择类型

```js
// helper-select/user.js

// 注册一个 用户选择 的辅助选择组件
import { registerComponent } from '@x-ui/otd-kit/shared';

registerComponent('user-select', (props, { attrs, slots }) =>
  h(
    XHelperSelect,
    {
      rowKey: 'id',
      titleKey: 'name',
      columns: [
        {
          title: '用户编号',
          dataIndex: 'id'
        },
        {
          title: '用户名',
          dataIndex: 'name',
          showInSearch: true
        },
        {
          title: '性别',
          dataIndex: 'gender'
        },
        {
          title: '年龄',
          dataIndex: 'age'
        }
      ],
      request: () => [
        { id: 1, name: '张三', gender: '男', age: 18, birthday: '2020-01-01' },
        { id: 2, name: '李四', gender: '女', age: 19, birthday: '2020-01-01' },
        { id: 3, name: '王五', gender: '男', age: 20, birthday: '2020-01-01' }
      ],
      ...attrs,
      ...props
    },
    slots
  )
);
```

### 路径别名配置

建议在 `vite.config.js` 中配置路径别名：

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import { resolve } from 'path';

export default defineConfig({
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'),
      '@/components': resolve(__dirname, 'src/components'),
      '@/utils': resolve(__dirname, 'src/utils'),
      '@/assets': resolve(__dirname, 'src/assets')
    }
  }
});
```

## 最佳实践

1. **保持目录层级简单**: 避免超过 4 层的深度嵌套
2. **文件数量控制**: 单个目录下文件数量不超过 20 个
3. **命名一致性**: 整个项目保持统一的命名风格
4. **功能内聚**: 相关功能的文件组织在一起
5. **职责清晰**: 每个目录都有明确的职责和用途
